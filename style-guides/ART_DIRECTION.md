## 2. 模块化时装

### 2.1 Slot 架构

角色采用模块化 slot 架构：剪影定义层 (hair/cloth/top) 决定角色轮廓；稀有度表达层 (ear/decorate/mask/bodyglow/tattoo) 区分品质与主题。

| Slot | 角色 |
|------|------|
| `hair` | 剪影定义 |
| `cloth` | 剪影定义 |
| `hat` | 稀有度表达 |
| `ear` | 稀有度表达 |
| `top` | 剪影定义 |
| `decorate` | 稀有度表达 |
| `mask` | 稀有度表达 |
| `bodyglow` | 稀有度表达 |
| `tattoo` | 稀有度表达 |

### 2.2 设计原则

- 付费时装按可替换 slot 组合设计，而非单张全身皮肤
- 剪影定义层 (hair/cloth/top) 决定角色轮廓
- 稀有度表达层 (ear/decorate/mask/bodyglow/tattoo) 区分品质与主题
- bodyglow、mask、tattoo 保留给高稀有度/付费视觉差异化
- 每套时装产出风格指南：剪影、材质、主题纹样、稀有度 VFX、色彩协调、UI 卡片展示

### 2.3 付费时装展示闭环

付费时装需要完整展示闭环：预览场景(previewScene) → 卡片背景(cardBg) → 揭示动效(revealMotion) → 稀有度特效(rarityVfx) → 商店入口(shopEntry) → 已拥有状态(ownedState)。

| 组件 |
|------|
| `previewScene` |
| `cardBg` |
| `revealMotion` |
| `rarityVfx` |
| `shopEntry` |
| `ownedState` |

## 3. 法宝 / 灵宝视觉四件套

法宝/灵宝需要视觉四件套：图标(icon) → 详情卡片(detailCard) → 实体预制体(entityPrefab) → 待机动效(idleMotion) → 技能特效(skillVFX) → 觉醒特效(awakenVFX)。

| 组件 |
|------|
| `icon` |
| `detailCard` |
| `entityPrefab` |
| `idleMotion` |
| `skillVFX` |
| `awakenVFX` |

### 设计原则

- 法宝按剪影和运动轨迹区分家族（飞剑/宝珠/幡旗/印玺/古宝），不仅靠颜色
- 法宝进阶支持：解锁、觉醒、技能预览、代币商店、槽位/装备状态
- 粒子与拖尾遮罩赋予法宝生命感，但保持 UI 卡片可读性
- 生成游戏中定义 ArtifactVisualSpec：图标、卡片边框、实体预制体、待机动效、技能 VFX、觉醒 VFX、稀有度调色板

## 4. 战斗 VFX 分类

战斗 VFX 分为七阶段管线：前摇提示(anticipation) → 弹道/拖尾(trail) → 命中冲击(impact) → 受击闪白(hitFlash) → 增益标记(buff) → 减益标记(debuff) → UI 数值反馈(uiNumberFeedback)。

| 阶段 |
|------|
| `anticipation` |
| `trail` |
| `impact` |
| `hitFlash` |
| `buff` |
| `debuff` |
| `uiNumberFeedback` |

### 设计原则

- 武器/技能线使用一致后缀：preview、skill、hit、trail、idle、awaken
- UI 奖励特效与战斗特效分离（时长、亮度、运动曲线不同）
- 受击特效保持短暂且保护剪影（移动端可读性）
- 大型 VFX 仅用于大招、觉醒或稀有掉落
- 每个战斗特性产出 GameFeelChecklist

## 5. ScenePaintingBrief 模板

高价值系统需要使用插画级过渡场景，而非功能菜单。过渡场景用于建立氛围和奖励幻想。

### 境界突破

| 属性 | 值 |
|------|-----|
| 主题 | 修仙者境界突破 |
| 氛围 | 震撼、升华、天地共鸣 |
| 镜头角度 | 仰角或平视主角全身 |
| UI 安全区 | 顶部 15% 留标题区，底部 10% 留确认按钮区 |
| 调色板 | 以主角属性色为主，环境色由暗转明 |
| 动画层 | 祥云汇聚、光柱冲天、境界文字浮现 |

### 秘境进入

| 属性 | 值 |
|------|-----|
| 主题 | 秘境入口开启 |
| 氛围 | 神秘、未知、探索欲 |
| 镜头角度 | 正面平视或轻微俯角展示入口全貌 |
| UI 安全区 | 中央 60% 留秘境名称和难度选择区 |
| 调色板 | 暗色基调配发光入口轮廓 |
| 动画层 | 入口光纹流转、粒子向内吸入 |

### Boss

| 属性 | 值 |
|------|-----|
| 主题 | Boss 登场 |
| 氛围 | 压迫感、威胁、史诗感 |
| 镜头角度 | 仰角强调 Boss 体型，主角在画面下方 1/3 |
| UI 安全区 | 顶部留 Boss 名称血条区 |
| 调色板 | 暗红/暗紫基调，Boss 局部高亮 |
| 动画层 | 屏幕震动、Boss 咆哮粒子、UI 血条渐显 |

### 时装展示

| 属性 | 值 |
|------|-----|
| 主题 | 时装/外观展示 |
| 氛围 | 优雅、华丽、收藏欲 |
| 镜头角度 | 360° 旋转展示台视角 |
| UI 安全区 | 中央主角展示区，底部购买/预览按钮区 |
| 调色板 | 深色背景突出时装主体，稀有度对应光效 |
| 动画层 | 角色缓慢旋转、时装部件依次高亮、稀有度边框呼吸 |

### 法宝觉醒

| 属性 | 值 |
|------|-----|
| 主题 | 法宝/灵宝觉醒 |
| 氛围 | 神圣、力量觉醒、蜕变 |
| 镜头角度 | 特写法宝主体，镜头由近拉远 |
| UI 安全区 | 顶部法宝名称，底部属性变化对比区 |
| 调色板 | 法宝属性色渐变，觉醒前后明暗对比 |
| 动画层 | 法宝破裂/重组动画、属性数值跳动、觉醒 VFX 爆发 |

### 活动开启

| 属性 | 值 |
|------|-----|
| 主题 | 限时活动开幕 |
| 氛围 | 热闹、紧迫、节日感 |
| 镜头角度 | 全景展示活动场景 |
| UI 安全区 | 中央活动标题，底部倒计时和入口按钮 |
| 调色板 | 活动主题色为主，明亮高饱和 |
| 动画层 | 烟花/花瓣粒子、倒计时数字跳动、奖励预览轮播 |

## 7. 反模式清单

- 禁止页游式金色描边按钮和廉价光效
- 禁止满屏飘字（移动端可读性优先）
- 禁止高饱和配色（修仙题材保持克制）
- 禁止超过 3 种主色同时出现
- 禁止用渐变色卡替代真实插画（高价值入口场景）
- 禁止在时装 UI 中使用普通背包页面的视觉框架
